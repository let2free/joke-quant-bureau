## 九大角色定义

### 角色1：指挥官（元宝/Commander）

| 项目 | 内容 |
|------|------|
| **职责** | 整体协调、任务分解、冲突仲裁、最终输出 |
| **输入** | 用户需求 + 各Agent输出 |
| **输出** | 最终ETF推荐清单 + 双轨融合报告 |
| **升级权限** | 可否决任何Agent的结论，需说明理由 |
| **触发条件** | 任何Phase开始时 |

#### 协作协议

```
1. 任务分解：
   - 收到用户需求 → 创建TaskList
   - 为每个Task分配Agent（按专长）
   - 设置Task依赖关系

2. 冲突仲裁：
   - 若两大Agent结论冲突 → 召集"争议解决会议"
   - 会议参与者：冲突双方 + 相关领域Agent + 指挥官
   - 仲裁依据：①历史准确率 ②最新数据 ③L1-L7框架符合度

3. 最终输出：
   - 汇总所有Agent输出
   - 标注"高置信度"（双轨一致）/"中等置信度"（有分歧）/"低置信度"（需人工复核）
```

### 角色8：因子研究员（因子迭代引擎）

| 项目 | 内容 |
|------|------|
| **职责** | 因子IC计算、权重迭代、新因子发现 |
| **输入** | 历史预测记录 + 实际涨跌数据 |
| **输出** | 因子权重调整建议 + 新因子候选清单 |
| **触发条件** | 预测准确度 <95% 时自动激活 |

#### 迭代流程

```
1. 计算各因子IC（信息系数）：
   IC = correlation(factor_score, actual_return)
   
2. 调整权重：
   - IC > 0.3 → 权重 +5%
   - IC 0.1~0.3 → 权重不变
   - IC < 0.1 → 权重 -5%，连续3次<0则剔除

3. 新因子发现：
   - 若某新因子IC > 0.3 → 提示人工审核后加入
   - 示例：发现"ETF份额变化"因子IC=0.35 → 建议加入
```

## 冲突解决机制

### 冲突类型1：Agent间结论冲突

| 冲突示例 | 解决机制 |
|-----------|---------|
| 大A推荐通信ETF #1，但量化1哥认为估值过高 | 指挥官召开"估值争议会"，包青天有最终否决权 |
| 老美认为FOMC偏鸽概率40%，但军机处认为鹰派风险更高 | 查阅历史准确率，准确率高的Agent权重更高 |
| Track A排名与Track B打分完全相反 | 标记为"低置信度"，建议用户等确认信号 |

### 冲突类型2：数据质量冲突

| 冲突示例 | 解决机制 |
|-----------|---------|
| WebSearch结果与TDX数据不一致 | 以TDX为准（实时行情），WebSearch仅作参考 |
| 北向资金数据混有历史数据 | 数据工程师标注"数据质量存疑"，降级使用 |

### 冲突类型3：因子权重冲突

| 冲突示例 | 解决机制 |
|-----------|---------|
| 因子研究员建议提高动量权重，但回测分析师发现近期衰减 | 以回测结果为准，暂缓权重调整 |
| 不同市场状态下最优权重不同 | 在市场状态自适应调整机制中处理 |

## 协作时间线（典型周度扫描）

```
D-1（准备日）：
  - 指挥官：创建TaskList，分配任务
  - 数据工程师：预拉取所需数据，检查质量

D-Day 09:00（执行日）：
  09:00 - 09:30  Phase 0：系统性因子预检（指挥官 + 军机处）
  09:30 - 10:30  Phase 1：并行侦察（大A + 老美 + 军机处）
  10:30 - 11:00  Phase 2：深度分析（量化1哥 + 数据工程师）
  11:00 - 11:20  Phase 3：交叉验证（因子研究员）
  11:20 - 11:40  Phase 4：风控审查（包青天）
  11:40 - 12:00  Phase 5：融合输出（指挥官 + Track B）

D-Day 15:30（收盘后）：
  - 数据工程师：拉取实际收盘数据
  - 因子研究员：计算准确度，若<95%触发复盘

D+1（复盘日）：
  - 全自动复盘（若设置了automation）
  - 或手动触发：复盘迭代流程R1-R6
```

## 注意事项

1. **指挥官有最终决定权**，但需听取专业Agent意见
2. **数据工程师是"数据守门人"** —— 所有数据需经其验证才能进入分析流程
3. **包青天有否决权** —— 任何ETF推荐若被风控否决，需写入`risk_log.md`并说明理由
4. **因子研究员自动触发** —— 预测准确度<95%时自动激活，无需人工介入
5. **所有Agent输出需标注置信度** —— 方便最终融合时判断权重
