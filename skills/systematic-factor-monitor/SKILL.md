# 系统性因子监测 Skill

在每次量化分析（周度ETF扫描、单次预测）**最开始**调用本 skill，完成 L0 + L4b 因子预检后，再进入 L1-L7 主流程。

## 调用时机

| 场景 | 是否调用 |
|------|---------|
| 周度ETF扫描（如"分析下周A股美股走势"） | ✅ 必须调用 |
| 单次ETF推荐 | ✅ 必须调用 |
| 复盘/验证准确度 | ✅ 必须调用（对比预测与实际） |
| 非量化类对话 | ❌ 不调用 |

## 执行流程

```
Phase 0: 系统性因子预检（本Skill核心）
  ├─ Step 1: L0 宏观流动性检测
  ├─ Step 2: L4b 微观结构冲击检测
  └─ Step 3: 因子权重输出 → 注入主分析

主分析（L1-L7，由九章量化局SOP执行）
  ├─ L1 赛道结构
  ├─ L2 竞争格局
  ├─ L3 护城河
  ├─ L4 供需缺口（+L4b修正）
  ├─ L5 催化剂
  ├─ L6 估值
  └─ L7 买入纪律

Phase R: 复盘迭代（预测失败后自动触发）
  ├─ R1: 拉取实际数据
  ├─ R2: 对比预测，计算准确度
  ├─ R3: 归因分析（哪个因子漏检/误判）
  ├─ R4: 升级检测逻辑（更新 references/ 文件）
  └─ R5: 调整因子权重（更新 SKILL.md 权重表）
```

## Step 1: L0 宏观流动性检测

### 1.1 检测信号清单

| 信号 | 数据来源 | 检测频率 | 冲击力度 | 检测方法 |
|------|---------|---------|---------|---------|
| 央行OMO/MLF操作量+利率 | 人民银行官网 / TDX | 每日09:00前 | ⭐⭐⭐⭐⭐ | WebSearch + TDX |
| DR007/R007（银行间利率） | 全国银行间同业拆借中心 | 实时 | ⭐⭐⭐⭐ | WebSearch |
| 北向资金连续流向 | TDX（869998） | 每日 | ⭐⭐⭐⭐ | TDX quotes |
| 央行沟通措辞变化 | 货币政策报告 / 新闻发布会 | 事件驱动 | ⭐⭐⭐⭐⭐ | WebSearch |
| 央行资产负债表变化 | 人行官网 | 月度 | ⭐⭐⭐ | WebSearch |

### 1.2 流动性状态判定标准

```
检测步骤：
1. 拉取近5日央行OMO/MLF操作公告
   - 若有操作 → 记录操作量、利率、期限
   - 计算"超预期程度" = (实际操作量 - 市场预期) / 市场预期
     （市场预期可参考Wind/同花顺分析师预期，若无则用前3月均值）

2. 拉取DR007前日收盘值 + 近5日趋势
   - DR007 ↓ → 流动性宽松信号
   - DR007 ↑ 且突破政策利率20bp → 收紧信号
   - 政策利率 = 7天逆回购利率（当前1.50%）

3. 拉取北向资金近5日累计净流入
   - 连续5日净流入 → "外资回流"标记 = 真
   - 累计金额 / A股自由流通市值 → 计算冲击系数
     冲击系数 > 0.5% → 强力看多信号
     冲击系数 0.2-0.5% → 中度看多
     冲击系数 < 0.2% → 弱信号

4. 综合判定流动性状态（三档）：
   - 🟢 宽松（Liquidity-Accommodative）
     OMO操作量超预期 AND (DR007下行 OR 北向连续净流入)
   - 🟡 中性（Liquidity-Neutral）
     上述条件不完全满足，但无收紧信号
   - 🔴 收紧（Liquidity-Tight）
     OMO操作量低于预期 AND DR007上行突破政策利率+20bp
```

### 1.3 流动性因子权重调整规则

|------------|----------------|----------------|------|--------|
| 🟢 宽松 | +10% | -5% | DR007突破政策利率时权重减半（v1.1修正） | v1.1 |

## Step 2: L4b 微观结构冲击检测

### 2.1 指数事件日历（未来30天）

**主要指数审核频率**：

| 指数系列 | 审核频率 | 生效月份 | 公告提前期 |
|---------|---------|---------|---------|
| 中证系列（沪深300/中证500/中证1000） | 每半年 | 6月、12月 | 约1-2个月 |
| 科创50 | 每季度 | 3月、6月、9月、12月 | 约1个月 |
| 创业板指 | 每季度 | 3月、6月、9月、12月 | 约1个月 |
| MSCI中国 | 每季度 | 2月、5月、8月、11月 | 约1个月 |
| 富时罗素 | 每季度 | 3月、6月、9月、12月 | 约1个月 |

**检测方法**：

```
1. 搜索"中证指数审核公告 2026"、"科创50样本调整 2026"等关键词
   数据源：中证指数官网、上交所官网、深交所官网、Wind/同花顺新闻

2. 若发现未来30天内有指数审核生效事件：
   a. 记录：指数名称、生效日期、新增/删除成分股列表
   b. 计算被动资金冲击系数：
      冲击系数 = |新权重 - 旧权重| × 跟踪该指数的基金规模 / 该股近20日日均成交额
      - 冲击系数 > 3 → 🔴 强烈买入/卖出信号
      - 冲击系数 1-3 → 🟡 中度信号
      - 冲击系数 < 1 → 🟢 弱信号
   c. 识别受益ETF：
      - 若成分股调整涉及半导体 → 半导体ETF受益
      - 若涉及科创50 → 科创50ETF受益
      - 若涉及创业板 → 创业板ETF受益

3. 输出"调仓窗口"标记：
   - 生效日前5个交易日 → 启动"调仓窗口"标记
   - 在ETF推荐中，对应ETF排序权重 +15%
```

### 2.2 被动资金流检测（每日）

```
检测步骤：
1. 检查未来5个交易日是否有指数审核生效
2. 若有 → 计算"被动资金流入压力"
   - 压力 = 跟踪资金规模 × 调仓比例 / 剩余交易日
   - 压力 > 1亿/日 → 强烈买入信号
3. 检查龙虎榜/大宗交易
   - 若连续3日出现同一ETF成分股的大宗交易 → "抢跑"信号
```

## 自动复盘迭代机制

### 触发条件

**每次预测结束后**，自动执行以下检查：

```
1. 等待市场收盘（A股15:00 / 美股次日05:00 EST）
2. 拉取实际数据（指数收盘涨跌幅、推荐ETF涨跌幅）
3. 对比预测值，计算准确度：
   - 方向准确度（预测涨/跌 vs 实际涨/跌）→ 目标 >95%
   - 幅度准确度（预测涨跌幅 vs 实际涨跌幅，误差<1%为准确）→ 目标 >80%
4. 若准确度 < 目标值 → 自动触发复盘迭代
```

### 复盘迭代流程

```
R1: 数据拉取
  - 使用TDX tool拉取实际收盘数据
  - 格式化：指数涨跌幅、ETF涨跌幅（与预测时相同格式）

R2: 准确度计算
  - 方向准确度 = (正确预测方向数 / 总预测数) × 100%
  - 幅度准确度 = (|预测涨跌幅 - 实际涨跌幅| < 1% 的数量 / 总预测数) × 100%
  - ETF排序准确度 = (预测排名 vs 实际涨幅排名) 的Spearman相关系数

R3: 归因分析
  - 哪个因子漏检？→ 更新 references/liquidity_detection.md
  - 哪个因子误判？→ 更新权重表
  - 是否有新因子未纳入？→ 扩展检测清单
  - ETF排序为何失真？→ 更新排序算法

R4: 升级检测逻辑
  - 将新发现的信号加入"检测信号清单"
  - 更新判定标准（如流动性状态判定阈值）
  - 更新被动资金冲击系数计算公式

R5: 调整因子权重
  - 根据误差方向，调整 L0/L4b 因子在排序中的权重
  - 更新 SKILL.md 中的"权重调整规则"表

R6: 输出复盘报告
  - 准确度评分卡
  - 误差根因分析
  - 升级内容清单
  - 下次预测注意事项
```

## 注意事项

1. **本Skill必须在主分析前调用** —— L0/L4b因子是"前提条件"，不是"事后解释"
2. **检测结果为"负"时也要报告** —— "流动性收紧"和"流动性宽松"同样重要
3. **复盘迭代必须执行** —— 每次预测准确度<95%时，必须触发R1-R6流程
4. **升级内容必须记录** —— 每次升级后，写入 `~/.workbuddy/memory/recap-log.md`，形成系统进化日志
5. **目标明确** —— ETF预测方向准确度 >95%，幅度准确度 >80%，排序Spearman系数 >0.85
