# AB并行量化分析

## 概述

本Skill实现AB双方案并行量化分析框架，同时执行两种分析方案并输出对比报告。

- **方案A**：Vibe-Trading CLI（AI驱动，多智能体协作）
- **方案B**：tdx-connector MCP + 九章量化局2.3（L0盘前信号 + L0.5催化剂 + L0.6美股联动 + L0.8风格轮动 + L1.2技术量化 + DAG并行 + 多空辩论 + 反身性检查 + 概率分布三场景）

## 触发条件

当用户提出以下意图时，使用本Skill：
- 「AB并行分析茅台」
- 「九章量化AB对比」
- 「用AB方案分析XX股票」
- 「对比Vibe和tdx分析结果」

## 执行流程

### 步骤1：解析股票代码

从用户输入中提取股票代码或名称，调用 `tdx-connector` 的 `tdx_lookup_stock` 解析。

### 步骤2：并行启动双方案

#### 方案B（九章量化局2.2，优先执行）

**B.0 盘前信号检查（L0，强制）**

1. 隔夜外盘：调用 `tdx_quotes` 获取美股指数（.DJI/.IXIC/.SOX）和A50期货
2. 亚太早盘：获取恒生/日经开盘
3. 重大消息：调用 `wenda_news_query` 当日新闻 + `wenda_macro_query` 宏观数据
4. 催化剂评分：对发现的催化剂按 V2.2 评分表打分
5. 反身性检查：连跌/连涨天数、一致性方向
6. 风格轮动信号：判断地缘/宏观事件对进攻vs防御资产的差异化影响

**B.1 数据获取**

1. 调用 `tdx-connector` 获取实时行情（`tdx_quotes`）
2. 调用 `tdx-connector` 获取K线数据（`tdx_kline`，周期=4，wantNum=60）
3. 按九章量化局2.1框架分析：
   - Layer 0：盘前信号 + 催化剂评分
   - Layer 1：DAG并行（技术指标 + 基本面 + 资金流向）
   - Layer 2：多空辩论（多头论点 vs 空头论点）
   - Layer 2.5：反身性检查（连跌天数 + 一致性评估）
   - Layer 3：风险评估（激进/中立/保守三档），Beta弹性分层校准
   - Layer 4：概率分布三场景（乐观/基准/悲观）+ 操作建议
4. 生成Markdown报告，保存到 `Claw/AB分析_方案B_<股票代码>_<日期>.md`

#### 方案A（后台执行，完成后通知）

1. 检查大模型V状态（调用 `vibe_service.ps1 status`）
2. 如果未启动，调用 `vibe_service.ps1 start` 启动
3. 后台执行Vibe-Trading分析：
   ```bash
   cd C:/Users/let2free/.vibe-trading
   vibe-trading.exe run -p "全量分析<股票名称>(<代码>)：技术分析(MACD/RSI/布林带)、基本面(PE/PB/ROE)、资金流向、风险评估，给出操作建议和目标价" --no-rich > Claw/AB分析_方案A_<股票代码>_<日期>.txt 2>&1
   ```
4. 监控进度，完成后读取输出文件并解析

### 步骤3：生成对比报告

汇总方案A和方案B的结果，输出对比表格：

| 对比维度 | 方案A（Vibe-Trading） | 方案B（tdx-connector） |
|----------|------------------------|--------------------------|
| 综合评级 |  |  |
| 目标价 |  |  |
| 操作建议 |  |  |
| 风险评级 |  |  |
| 分析深度 |  |  |
| 数据质量 |  |  |

### 步骤4：输出与归档

1. 完整报告保存到 `Claw/AB分析_对比报告_<股票代码>_<日期>.md`
2. 同步到IMA知识库（可选，需用户确认）
3. 返回核心结论到对话

## 依赖检查

执行前检查以下依赖：

| 依赖 | 检查方式 | 缺失处理 |
|------|---------|---------|
| tdx-connector MCP | 调用 `tdx_quotes` 测试 | 提示用户配置 |
| Vibe-Trading CLI | 检查 `vibe-trading.exe` 是否存在 | 提示用户安装 |
| 大模型V服务 | 调用 `vibe_service.ps1 status` | 自动启动 |

## 注意事项

1. **方案A可能卡住**：Vibe-Trading CLI可能因网络问题卡住，设置超时（建议300秒）并给出降级方案
2. **数据一致性**：方案A和方案B使用的数据源不同，对比时需注意数据日期差异
3. **大模型V状态**：如果大模型V未启动，先调用 `vibe_service.ps1 start` 启动
4. **输出文件管理**：定期清理 `Claw/` 目录下的历史分析报告
