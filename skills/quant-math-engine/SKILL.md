# 量化数学因子引擎（Track B）

## 调用时机

以下任一条件触发时，调用本 Skill：

1. 用户说"分析下周走势"/"推荐 ETF"—— **与 Track A（AI体系）并行执行**
2. 用户说"数学因子打分"/"Track B 分析"
3. 复盘时发现 ETF 排名准确度 <95%—— **触发因子权重自动迭代**

---

## 执行流程

### Phase 1 — 因子计算

对每只候选 ETF 计算以下因子（默认回望窗口：1W/1M/3M）：

#### 第一类：动量类因子

| 因子名 | 公式 | 说明 |
|--------|------|------|
| `mom_1w` | `ret_5d = (P_t / P_{t-5}) - 1` | 1周动量 |
| `mom_1m` | `ret_20d = (P_t / P_{t-20}) - 1` | 1月动量 |
| `mom_3m` | `ret_60d = (P_t / P_{t-60}) - 1` | 3月动量（核心） |
| `mom_6m` | `ret_120d` | 6月动量（长期确认） |
| `risk_adj_mom` | `ret_60d / vol_60d` | 风险调整动量（夏普类） |
| `dual_mom` | `ret_60d if ret_120d>0 else -999` | 双重动量（绝对+相对） |

#### 第二类：均值回归类因子

| 因子名 | 公式 | 说明 |
|--------|------|------|
| `dist_from_ma20` | `(P_t - MA20) / MA20` | 偏离20日均线（负=超卖） |
| `dist_from_ma60` | `(P_t - MA60) / MA60` | 偏离60日均线 |
| `boll_pos` | `(P_t - lower) / (upper - lower)` | BOLL带位置（0=下轨，1=上轨） |
| `rsi_14` | `RSI(14)` | RSI超买超卖信号 |

#### 第三类：波动率类因子

| 因子名 | 公式 | 说明 |
|--------|------|------|
| `vol_20d` | `std(ret_daily, 20)` | 20日历史波动率（越低越好） |
| `vol_ratio` | `vol_recent / vol_historical` | 波动率放大因子（>1.2=放大） |
| `max_dd_20d` | `max drawdown over 20d` | 20日最大回撤（越低越好） |

#### 第四类：成交量/资金流类因子

| 因子名 | 公式 | 说明 |
|--------|------|------|
| `vol_ratio_5d` | `vol_t / avg(vol, 5)` | 量比（当日/5日均） |
| `fund_flow_5d` | `sum(inflow, 5) / avg(sum(inflow, 5), 20)` | 资金流入强度 |
| `northbound_5d` | `北向资金5日累计 / AUM` | 北向资金流入强度（A股专属） |
| `etf_flow_5d` | `ETF净申购额5日累计 / AUM` | ETF份额变动（机构 vs 散户） |

#### 第五类：微观结构类因子

| 因子名 | 公式 | 说明 |
|--------|------|------|
| `index_rebalance` | `(new_w - old_w) × AUM / adv` | 指数调仓冲击系数（>3=强烈） |
| `premium` | `(ETF价格 - NAV) / NAV` | ETF溢价率（高=谨慎） |
| `turnover` | `vol / shares_outstanding` | 换手率（高=情绪热） |

#### 第六类：市场状态类因子（用于动态调整权重）

| 因子名 | 计算公式 | 状态输出 |
|--------|----------|----------|
| `market_regime` | `MA20 vs MA60 + vol_20d vs vol_60d` | 进攻/中性/防守 |
| `vix_level` | `VIX index level` | 低波动/正常/高波动 |
| `breadth` | `% of stocks above 20MA` | 市场宽度（高=健康） |

---

### Phase 2 — 因子打分与合成

#### Step 2.1：因子归一化

所有因子统一映射到 **0-100 分**：

```
# 动量类：越高越好
score = (rank - min_rank) / (max_rank - min_rank) × 100

# 均值回归类：负偏离越大分越高（超卖=高分）
score = -(dist_from_ma) 再归一化

# 波动率类：越低越好（反转）
score = 100 - normalized_vol

# 资金流类：越高越好
score = min(normalized_flow, 100)
```

#### Step 2.2：加权合成总分

```
Total_Score_i = w1×mom_score + w2×mr_score + w3×vol_score 
               + w4×flow_score + w5×ms_score + w6×regime_adj
```

**默认权重（v1.0）**：

| 权重 | 因子类别 | 默认值 | 调整逻辑 |
|--------|----------|--------|----------|
| w1 | 动量类 | 35% | 趋势市提高，震荡市降低 |
| w2 | 均值回归类 | 15% | 震荡市提高，趋势市降低 |
| w3 | 波动率/风险类 | 10% | 高波动市场提高 |
| w4 | 资金流类 | 25% | A股北向资金权重可单独提高 |
| w5 | 微观结构类 | 10% | 指数调仓窗口期提高至25% |
| w6 | 市场状态调节 | ±10% | 进攻偏动量，防守偏低波 |

#### Step 2.3：市场状态自适应调整（重要！）

```
if market_regime == "进攻":
    w1 += 10%, w2 -= 5%, w3 -= 5%
elif market_regime == "防守":
    w1 -= 10%, w2 += 5%, w3 += 5%, w4 += 5%
else:  # 中性
    保持默认权重
```

## 因子迭代机制

### 触发条件

每次预测后，计算以下准确度指标：

| 指标 | 计算公式 | 目标值 | 触发迭代阈值 |
|------|----------|--------|--------------|
| 方向准确度 | `pred_dir == actual_dir` 的准确率 | >95% | <95% |
| 幅度准确度 | `1 - |pred_ret - actual_ret| / |actual_ret|` 的平均值 | >80% | <80% |
| 排名相关性 | `Spearman(rank_pred, rank_actual)` | >0.85 | <0.85 |

### 迭代动作

当准确度 < 目标值时，自动执行：

1. **因子权重调整**：用最近3次预测的实际数据，计算各因子 IC（信息系数），提高高 IC 因子权重
2. **因子剔除**：若某因子连续3次 IC < 0，自动剔除
3. **新因子加入**：若某新因子 IC > 0.3，提示人工审核后加入
4. **市场状态权重更新**：若某市场状态下准确度显著偏低，调整该状态下的权重分配公式

## 数据获取指引

| 因子类别 | 主要数据源 | TDX工具调用 |
|----------|----------|-------------|
| 价格动量 | 历史收盘价 | `tdx_kline` |
| 波动率 | 历史收盘价 | `tdx_kline` → 计算std |
| 成交量 | 历史成交量 | `tdx_kline` |
| 北向资金 | 北向资金流向 | `tdx_screener` 或 WebSearch |
| ETF份额 | 基金份额变动 | `tdx_quotes` → 基金份额字段 |
| 指数调仓 | 指数公司公告 | WebSearch |
| 市场状态 | 主要指数 MA + VIX | `tdx_quotes` + WebSearch |
